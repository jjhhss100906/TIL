# Git이란?

Git이란 **버전 관리 시스템**입니다.2005년 출시 이후 5가지 목표를 잃지 않고 지금까지 운영되고 있다. 5가지 목표는 뭐냐면

* 빠른 속도
* 단순한 구조
* 비선형적인 개발
* 완벽한 분산
* 대형 프로젝트에도 유용할것

## Git을 사용하는 이유

Git을 이용하면 **쉬운 버전 관리**가 가능하다.프로잭트를 진행하면 잦은 업데이트가 필수적인데,프로젝트를 업대이트 할때마다 따로 저장하는 것이 아닌,Git으로 버전관리가 가능하다.그리고 쉽게 이전으로 돌아갈수있다.

여러 개발자들의 협업시 한 코드를 가지고 **병렬적으로 진행**하는것을 가능하게 해준다.

Git은 분산 관리 시스템이기 때문에 중앙 서버 없이 프로젝트 파일을 저장, 공유할수 있다.

## Git의 기본 용어 정리

* Repository:소스코드들이 저장되어 있는 물리적인 공간을 말한다.저장소에는 **로컬 저장소**와 **원격 저장소**2가지 종류가 있다.
로컬 저장소는 내가 직접 작업하는공간이고,
인터넷이 없어도 사용이 가능하다.   
그리고 원격 저장소는 인터넷에 있는 저장소이고,여러사람이 같이 쓰는 공간이다.
대표적으로 Github가 있다.

* Working Tree :현재 파일 수정,저장 등의 작업을 하는 디렉터라를 말한다.

* Index:commit을 실행하기 전의 저장소와 Working tree사이에 존재하는 공간을 말합니다.
저장소에 커밋하기 전에 커밋을 준비하는위치이다.커밋은 현재 파일 상태를 저장하는것이다.

* Head:현재 작업중인 브랜치를 말한다.
여기서 브랜치는 메인 코드에 영향을 주지않고,
새로운 기능이나 실험을 따로 작업이 가능하다.
