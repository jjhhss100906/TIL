# Git이란?

Git이란 **버전 관리 시스템**입니다.2005년 출시 이후 5가지 목표를 잃지 않고 지금까지 운영되고 있다. 5가지 목표는 뭐냐면

* 빠른 속도
* 단순한 구조
* 비선형적인 개발
* 완벽한 분산
* 대형 프로젝트에도 유용할것

## Git을 사용하는 이유

Git을 이용하면 **쉬운 버전 관리**가 가능하다.프로잭트를 진행하면 잦은 업데이트가 필수적인데,프로젝트를 업대이트 할때마다 따로 저장하는 것이 아닌,Git으로 버전관리가 가능하다.그리고 쉽게 이전으로 돌아갈수있다.

여러 개발자들의 협업시 한 코드를 가지고 **병렬적으로 진행**하는것을 가능하게 해준다.

Git은 분산 관리 시스템이기 때문에 중앙 서버 없이 프로젝트 파일을 저장, 공유할수 있다.

## Git의 기본 용어 정리

* Repository:소스코드들이 저장되어 있는 물리적인 공간을 말한다.저장소에는 **로컬 저장소**와 **원격 저장소**2가지 종류가 있다.
로컬 저장소는 내가 직접 작업하는공간이고,
인터넷이 없어도 사용이 가능하다.   
그리고 원격 저장소는 인터넷에 있는 저장소이고,여러사람이 같이 쓰는 공간이다.
대표적으로 Github가 있다.

* Working Tree :현재 파일 수정,저장 등의 작업을 하는 디렉터라를 말한다.

* Index:commit을 실행하기 전의 저장소와 Working tree사이에 존재하는 공간을 말합니다.
저장소에 커밋하기 전에 커밋을 준비하는위치이다.커밋은 **현재 파일 상태를 저장**하는것이다.

* Head:현재 작업중인 브랜치를 말한다.
여기서 브랜치는 **메인 코드에 영향을 주지않고,새로운 기능이나 실험을 따로 작업이 가능한것**을 뜻한다.

* Snapshot:특정 시점에서 파일,폴더 또는 워크스페이스의 상태를 의미한다.   
워크스페이스란 **실제로 파일을 보고 수정하는 공간**이다.

* Commit:변경된 작업 과정들의 대한 점검을 마치고 저장소에 남기는 역할을 한다.

* Checkout: **특정 시점이나 branch의 소스 코드로 이동하는것**을 의미한다. 이전 버전 작업을 불러오는데 많이 사용한다.

* Branch: Commit단위로 구분된 소스 코드 타임라인에서 분기점을 의미한다.Branch에서 작업을 완료하면,Merge의 작업을 수행할수있다.

* Merge: Branch와 Cranch의 내용을 합치는 작업인 병합을 의미한다.

## Git local,remote 개념
1. 로컬 저장소
    * 로컬 저장소는 개발자의 **컴퓨터 내**에 위치한  저장소이다.

    * 프로젝트 디렉토리 내에```.git```이라는 숨겨진 폴더가 생성되며,이 폴더안에 변경 이력,브랜치,설정 등이 저장된다.

    *  로컬 저장소는 개발자가 프로젝트를 **관리하고 수정**할때 사용하고 있다.
2. 원격 저장소
    * 원격 저장소는 인터넷이나 **네트워크 상의 다름 위치**에 위치한 저장소이다.

    * GitHub,GitLab,Bitbucket 등에 프로젝트를 **업로드하고 공유**하기위해 사용된다.

    * 여러 개발자가 협업할때 중앙 저장소의 역할을 하며, 여러 사용자 간에 코드를 공유하고 동기화하는데 사용된다.

로컬 저장소와 원격 저장소간의 차이

* 저장 위치
    * 로컬저징소는 **프로젝트 디렉토리 내**에 있다.
    * 원격 저장소는 네트워크상의 다른위치에 위치하고 있다.
* 저장 목적
    * 로컬 저장소는 개발자가 **코드를 수정**하고 **변경이력을 관리**하기 위해 사용한다.
    * 원격 저장소는 여러 개발자가 협력하고 코드를 공유하며,프로젝트를 백업하고 외부에 배포하기 위해 사용한다.
* 저장 내용
    * 로컬 저장소는 개발자의 **작업 디렉토리**와 **Git 이력**을 포함한다.
    * 원격 저장소는 로컬 저장소의 특정 시점을 포함하며,다수 사용자간의 코드 공유 및 협업을 지원한다.
