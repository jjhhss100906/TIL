# Github란?

Github는 크게 두가지 측면애서 이해할수있다.

#### Git저장소 호스팅 서비스


Githun의 기본기능은 Git저장소를 클라우드에 저장하고 관리하는것이다.로컬에서 Git으로 관리하던 코드를 Github에 업로드하면 다른 사람과 공유할수있다. 무료 계정으로도 무제한의 공개,비공개 저장소를 만들수있다.

#### 협업 플렛폼
Github는 단순한 저장소를 넘어서 개발자들이 함께일할수있는 다양한 도구를 제공한다.

* Pull Request:코드 변경사항을 제안하고 리뷰를받을 수 있다.
* Issues:버그 추적,기능 요청,작업 관리를 할수있다.
* Projects:칸반 보드로 작업을 시각화하고 관리가 가능하다.칸반 보드란 간단히 말하면 해야 할 일을 카드처럼 만들어서 단계별로 옮기는 보드이다.
* Discussions:프로젝트에 대한 질문과 논의를 할수있다.
* Github Actions:CI/CD 파이프라인을 구축할수있다.
CI/CD 파이프라인이란,일단 CI는 지속적 통합을 뜻하고 CD는 지속적 배포를 뜻한다.그리고 CI/CD 파이프라인은 코드를 만들고-테스트하고-배포하는
과정을 **자동으로 이어서 처리해주는 시스템**이다.
* Github Pages:정적 웹사이트를 무료로 호스팅 할수있다.**즉,HTML,CSS,JS 파일을 깃허브에 올리면 알아서 웹사이트를 만들어 주소까지 주는것을 말한다.** 

## Github와 Git의 관계

Git과 Github는 서로 다른것이다.

* Git:버전 관리 시스템
* Github:Git 저장소 호스팅 서비스

Git은 로컬에서 독립적으로 사용할수있으며,Github없이도 완전히 작동한다.Github는 Git을 기반으로 하지만,웹 인터페이스와 협업 기능을 추가로 제공한다.

## Github의 대안들

Github 외에도 비슷한 서비스들이 있다.

* Gitlab:오픈소스이며 자체 호스팅이 가능하다.
* Bitbucket:Atlassian 제품군과 잘 통합된다.여기서 Atlassian이란 **협업,프로젝트관리,개발 관리 등을 도와주는 소프트웨어 도구 모음**이라고 볼수있다.
* Azure Devops:Microsoft의 **개발 협업+ 프로젝트 관리 + 배포 자동화 도구 모음**.
* Gitea:자신이 직접 서버에 설치해서 사용하는 **깃 저장소 관리 서비스**이다.

이들은 기본적으로 유사한 기능을 제공하지만,Github는 가장 큰 커뮤니티와 생태계를 가지고있다. 특히 오픈소스 프로젝트의 대부분이 Github에서 호스팅 되고 있어서,오픈소스애 기여하거나, 다른 개발자들과 네트워킹 하기에 좋은 플렛폼이다.

## Github의 영향력

Github는 현대 소프트웨어 개발 문화에 큰 영향을 미쳤다.

#### 오픈소스 생태계의 중심이다.

전세계 수백만개의 오픈소스 프로젝트가 Github에서 관리되고 있다.Linux 커널,Python,React,TensorFlow 등 주요 오픈소스 프로잭트들이 모두 Github를 사용한다.

#### 협업 방식의 표준화

Pull Request(코드를 메인 코드에 넣어달라고하는 협업 방식)와 코드 리뷰(다른 사람이 자신의 코드를 확인하고 피드백 해주는것)는 Github가 대중화시킨 협업 방식이다.

#### 개발자 포트폴리오

Github 프로필은 개발자의 포트폴리오로 활용된다. 많은 기업이 채용 과정에서 지원자의 Github활동을 참고한다.
