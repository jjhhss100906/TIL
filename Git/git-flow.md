# Gitflow란?

## Gitflow의 개념

Gitflow는 Git으로 형성관리를 할때 브랜치를 효율적으로 관리하기 위해 사용하는 **브랜치 관리 전략**입니다.즉,여러 사람이 동시에 작업해도 코드가 꼬이지 않게 브랜치를 어떻게 나누고,합치고,운영할지 정하는 규칙이다.

## Gitflow 사용 이유

만약 프로젝트의 규모가 작거나 혼자서 개발을 할경우 그냥 작업해서 배포해도 상관이 없을 것이다. 하지만 규모가 커져서 팀원이 많아지면 충돌이 생기고 이슈가 터지면 개발을 멈춰야 하는 불편함이 있다.그래서 이러한 현상들을 효율적으로 하기위헤 생겨난 전략이라고 볼수있다.

## Gitflow 살펴보기

Git branch의 종류는 아래 다섯가지가 존재합니다

* master:제품으로 출시될수 있는 브랜치
* develop:다음 출시 버전을 개발하는 브랜치
* feature:기능을 개발하는 브랜치
* release:이번 출시 버전을 준비하는 브랜치
* hotfix:줄시 버전에서 발생한 버그를 수정 하는 브랜치
